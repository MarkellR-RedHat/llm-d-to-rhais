What You Get: Upstream vs Enterprise

llm-d is excellent software. This comparison helps decide when enterprise features justify the investment.

Upstream llm-d

  • Community forums and GitHub issues
  • No SLA or response time guarantees
  • Community-tested GPU support
  • Community security patches, no CVE SLA
  • Manual OpenShift integration
  • Self-paced learning, community meetups
  • Community-maintained docs and wikis

Enterprise Red Hat AI

  • 24/7 enterprise support with 1-4hr Sev-1 SLA
  • Dedicated Technical Account Manager
  • Red Hat certified GPU matrix (NVIDIA, AMD, Intel)
  • CVE patching within 24-48 hours, FIPS 140-2
  • Native OpenShift integration, Operator managed
  • Instructor-led training, Red Hat certification
  • Professional docs, reference architectures

Deploying Granite 3.3 8B

Same model, same parameters. The enterprise layer adds security, monitoring, and signed images.

llm-d (Upstream) YAML
apiVersion: llm-d.ai/v1
kind: Model
metadata:
name: granite-8b
spec:
model: ibm/granite-3.3-8b-instruct
replicas: 2
resources:
gpu: 1
routing:
strategy: kv-cache-aware
inference:
maxBatchSize: 128
tensorParallel: 1
Red Hat AI Inference Server YAML
apiVersion: ai.redhat.com/v1
kind: LLMInferenceService
metadata:
name: granite-8b
spec:
model: ibm/granite-3.3-8b-instruct
replicas: 2
resources:
gpu: 1
routing:
strategy: kv-cache-aware
inference:
maxBatchSize: 128
tensorParallel: 1
security:
mtls: enabled
rbac: enabled
monitoring:
auditLogging: enabled
image:
registry: registry.redhat.io
signed: true
